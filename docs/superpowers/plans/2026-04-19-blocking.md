# Blocking Feature Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add block person/community functionality behind a hamburger menu on profile and community pages, with a confirmation panel and toast on success.

**Architecture:** New `blockPerson`/`blockCommunity` API functions; `CommunityHeader` gets a Block button in its existing menu grid + confirmation panel; `ProfilePage` gets a new hamburger menu matching the same pattern; `FeedStack` wires the community block and reads navigation toast state on mount.

**Tech Stack:** React 18, TypeScript, lemmy-js-client, react-router-dom, Vitest + @testing-library/react

```ts
export async function fetchPersonDetails(
  instance: string,
  token: string | undefined,
  username: string,
  page: number,
): Promise<{ posts: PostView[]; comments: CommentView[]; personId: number | null }> {
  const res = await client(instance, token).getPersonDetails({
    username,
    sort: 'New',
    page,
    limit: 20,
  });
  return {
    posts: res.posts,
    comments: res.comments,
    personId: res.person_view?.person?.id ?? null,
  };
}
```
